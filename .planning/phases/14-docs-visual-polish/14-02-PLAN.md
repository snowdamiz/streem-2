  <action>
Update the inline `<style>` block in DocsApp.tsx to improve mobile layout for 375px viewports. The existing 700px breakpoint is a good start but needs these fixes:

1. **Sidebar becomes a horizontal scrollable nav bar on mobile** — avoids eating vertical space with a full-height nav. On mobile, the nav should:
   - Lay out horizontally: `display: flex; flex-direction: row; align-items: center; flex-wrap: wrap; gap: 8px`
   - Use `overflow-x: auto` to allow scrolling if nav items overflow
   - Have `padding: 12px 16px` (compact)
   - Keep `border-right: none` and add `border-bottom: 1px solid var(--color-border)` instead

2. **Logo stays visible on mobile** — Give `.docs-nav-brand` `flex-shrink: 0` so it doesn't compress when nav items wrap.

3. **Nav list becomes horizontal on mobile**:
   ```css
   .docs-nav-list { flex-direction: row; flex-wrap: wrap; gap: 4px 8px; }
   ```

4. **Content padding on mobile** — Reduce to `padding: 24px 16px` (from 24px 20px) to give more room on 375px. Set `min-width: 0` on `.docs-main` to prevent grid blowout.

5. **Code blocks on mobile** — Ensure `pre` elements don't cause horizontal overflow. The global.css already has `overflow-x: auto` on `pre`, but add to `.doc-pre` in the inline style:
   ```css
   .doc-pre { max-width: 100%; }
   ```

6. **Section headings** — On mobile, reduce `.docs-title` from `2.5rem` to `1.8rem`:
   ```css
   @media (max-width: 500px) {
     .docs-title { font-size: 1.8rem; }
   }
   ```

Keep the 700px breakpoint. Add a tighter 500px breakpoint for the title size fix.

Verify no `background: white` or `color: black` or any light-colored backgrounds remain anywhere in the inline styles.
