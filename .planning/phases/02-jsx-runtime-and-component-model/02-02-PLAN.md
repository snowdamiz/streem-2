<objective>
Implement reactive DOM bindings — the mechanism that makes JSX expressions containing signal accessor functions update individual DOM nodes surgically. This delivers JSX-02: signal reads in JSX update only the exact affected DOM node, no full component re-render.

Purpose: Without reactive bindings, JSX is static HTML generation. With them, `{() => count()}` in JSX creates an effect that updates exactly one text node when count changes. This is the core DX difference between Streem and React.

Output: bindings.ts with six binding functions; h.ts applyProps() extended with reactive dispatch; TDD test suite proving surgical DOM updates.
</objective>

<feature>
  <name>Reactive DOM Bindings</name>
  <files>packages/dom/src/bindings.ts, packages/dom/src/h.ts, packages/dom/tests/bindings.test.ts</files>
  <behavior>
    Six binding functions, each accepting a DOM node and an accessor `() => T`:

    bindTextNode(parent, accessor, anchor?) — creates text node, sets nodeValue = String(accessor()) inside effect
    Cases:
      signal value "hello" → text node contains "hello"
      signal changes to "world" → same text node now contains "world" (no new node created)

    bindAttr(el, name, accessor) — sets/removes attribute inside effect
    Cases:
      accessor returns "foo" → setAttribute(name, "foo")
      accessor returns null/undefined/false → removeAttribute(name)
      accessor returns true → setAttribute(name, name) (boolean attribute)

    bindClass(el, accessor: () => string) — sets el.className
    Cases:
      accessor returns "active" → el.className === "active"
      accessor returns "active selected" → el.className === "active selected"

    bindClassList(el, accessor: () => Record<string, boolean>) — toggles per-class
    Cases:
      {active: true, hidden: false} → classList.contains("active") true, "hidden" false
      update to {active: false, hidden: true} → classList toggled correctly

    bindStyle(el, accessor: () => Partial<CSSStyleDeclaration>) — Object.assign to el.style
    Cases:
      {color: "red"} → el.style.color === "red"

    bindEvent(el, eventName, handler) — addEventListener once, onCleanup removes
    Cases:
      click fires → handler called
      owner disposed → listener removed

    applyProps dispatch (in h.ts):
    - key starts with "on" + value is function → bindEvent
    - key === "class" + value is function → bindClass; static string → el.className = val
    - key === "classList" → bindClassList (function or object wrapped in () => obj)
    - key === "style" + value is object/function → bindStyle
    - value is function (anything else) → bindAttr reactive
    - value is static → setAttribute (existing behavior)
    - key === "ref" → call ref(el) (existing behavior)
    - key === "children" → skip (existing behavior)
  </behavior>
  <implementation>
    Write failing tests first (RED) covering all six binding types and the applyProps dispatch.
    Then implement bindings.ts and extend h.ts applyProps() to pass all tests (GREEN).
    Each binding wraps its update in effect() from @streem/core — effects auto-dispose when
    the parent createRoot scope disposes, leveraging the existing owner tree from Phase 1.

    **CRITICAL pattern from RESEARCH.md:**
    The accessor pattern means JSX expressions like `{() => count()}` are passed to h() as
    a function child. h() checks: if typeof child === 'function', create a text node and
    call bindTextNode(parent, child). For attrs: if typeof value === 'function', call bindAttr.
    This check in applyProps() is the dispatch mechanism.

    **Pitfall to avoid:**
    Do NOT call the accessor in applyProps before deciding whether it's reactive.
    `typeof value === 'function'` MUST be checked before ANY invocation of value.
    Calling value() in applyProps would consume the value as a snapshot, breaking reactivity.

    Also extend h() to handle function children (not just element children):
    In appendChildren(), if child is a function (typeof child === 'function'), treat it as
    a text accessor: create text node, call bindTextNode(parent, child, anchor).
  </implementation>
</feature>
