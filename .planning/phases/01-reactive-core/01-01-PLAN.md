    Implement the two internal foundation modules. These are NOT exported in the public API — they are internal implementation details that signal.ts (Plan 01-02) builds on top of.

    **packages/core/src/reactive.ts** — Push-pull reactive graph

    Implement the push-pull algorithm with these characteristics:
    - Source signals eagerly push dirty notifications to all subscribers when their value changes
    - Computed values are lazy: marked Pending on dirty notification, re-evaluated only when read
    - Effects are eager: re-run synchronously when notified dirty
    - Use Sets for `deps` and `subs` (O(1) deduplication; simplifies cleanup; see research Open Question 2)
    - Include a `currentSubscriber` module-level variable (the active tracking context)
    - Include an `isBatching` flag + deferred notification queue (stub for Phase 3 `batch()` — do not export publicly yet, just add the mechanism so Phase 3 can enable it without touching the algorithm)

    Exports needed by owner.ts and signal.ts:
    - `getCurrentSubscriber(): Subscriber | null` — for DX-02 warning check
    - `createSignalNode<T>(value: T): SignalNode<T>` — creates an internal signal node
    - `createComputedNode<T>(fn: () => T): ComputedNode<T>` — creates a lazy computed node
    - `createEffectNode(fn: () => void): EffectNode` — creates an eager effect node
    - `trackRead(source: SourceNode): void` — registers current subscriber as reader
    - `notifySubscribers(source: SourceNode): void` — pushes dirty to all subscribers

    Key implementation rules:
    1. Effect re-run MUST clear stale deps before re-tracking (see research Pitfall 1 — "Ghost subscriptions"). Before re-running: remove effect from all sources in its deps set, clear deps set, then run fn (which re-tracks).
    2. Circular dependency detection: add a `computing` flag to ComputedNode. If a computed is read while its `computing` flag is true, throw in dev (`import.meta.env.DEV`), return cached value in prod. See research Pitfall 4.
    3. Synchronous mutual notification guard: add a `notifying` flag to prevent re-entrant notification. If `notifySubscribers` is called while `notifying` is true, queue the source for notification after the current loop completes. This prevents double-firing (research Pitfall, anti-pattern 3).

    The `EffectNode` must store a reference to the owner it was created under so owner disposal can unregister it.

    **packages/core/src/owner.ts** — Owner tree

    Implement the owner tree exactly per CONTEXT.md locked decisions:

    ```
    interface Owner {
      parent: Owner | null
      children: Owner[]
      cleanups: (() => void)[]
      disposed: boolean
    }
    ```

    `createRoot<T>(fn: (dispose: () => void) => T): T`:
    - Creates a new Owner node
    - If currentOwner != null, adds new owner to parent's children array
    - Sets currentOwner = new owner during fn() execution
    - Returns fn's return value (which is the caller's reactive setup)
    - The dispose function: calls disposeOwner(owner) — bottom-up traversal (children first, then cleanups)

    `onCleanup(fn: () => void)`:
    - If currentOwner != null: push fn to currentOwner.cleanups
    - If currentOwner is null: fn is silently dropped (never called) — this is acceptable per research

    `getOwner(): Owner | null`:
    - Returns currentOwner

    `runWithOwner<T>(owner: Owner | null, fn: () => T): T`:
    - If `import.meta.env.DEV` and owner?.disposed: throw `new Error('[Streem] runWithOwner called with a disposed owner. The owner has already been disposed and can no longer accept new reactive computations.')`
    - In prod (or if owner is not disposed): set currentOwner = owner, run fn(), restore currentOwner, return result
    - Always restore currentOwner in a finally block

    `disposeOwner(owner: Owner)` (internal, not exported):
    - Guard: if owner.disposed return early
    - Set owner.disposed = true
    - Dispose children first (bottom-up): for child of owner.children, call disposeOwner(child)
    - Then fire own cleanups: for cleanup of owner.cleanups, call cleanup()
    - Clear: owner.children = [], owner.cleanups = []

    Export: `createRoot`, `onCleanup`, `getOwner`, `runWithOwner`
    Also export the `Owner` type (needed by signal.ts for getOwner() return type)
