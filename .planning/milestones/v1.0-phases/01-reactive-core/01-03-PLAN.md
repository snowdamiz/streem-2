  <behavior>
    All behaviors have exact I/O contracts — ideal for TDD.

    **SIGNAL-01 (signal.test.ts):**
    - `signal(0)()` returns `0`
    - `signal(0).set(5)` then `count()` returns `5`
    - TypeScript: `signal<string>('hello')` accepts only string values to `.set()`

    **SIGNAL-02 (signal.test.ts):**
    - `computed(() => count() * 2)` returns `0` when count = 0
    - After `count.set(5)`, `doubled()` returns `10` (lazy re-evaluation on read)
    - Computed with no dependency changes: cached value returned (no re-computation)
    - Diamond dependency: A → B, A → C, B+C → D — D evaluates exactly once per A change

    **SIGNAL-03 (signal.test.ts):**
    - `effect(() => { log.push(count()) })` runs immediately: log = [0]
    - After `count.set(1)`: log = [0, 1] (auto-tracked)
    - After `count.set(1)` again (same value): log still = [0, 1] (Object.is equality, no re-run)
    - Effect with conditional read: if count() > 0, reads nameSignal(); after count goes 0→1, effect subscribes to nameSignal; after count goes 1→0, effect unsubscribes from nameSignal (stale dep cleanup)
    - Effect dispose function: calling dispose() stops re-runs

    **SIGNAL-04 (owner.test.ts):**
    - `createRoot(dispose => { ... })` — effects inside run initially
    - After `dispose()`, signal changes no longer trigger inner effects
    - Nested `createRoot` inside outer: disposing outer also disposes inner (bottom-up)

    **SIGNAL-05 (owner.test.ts):**
    - `onCleanup(cb)` inside effect: cb fires before each re-run, then again on scope disposal
    - `onCleanup(cb)` inside `createRoot`: cb fires once on dispose
    - Effect cleanup order: multiple `onCleanup` calls → all fire on re-run (in registration order)
    - Cleanup fires before new effect run: old cleanup fires, THEN new run executes

    **DX-02 (dev-warnings.test.ts):**
    - `signal(0, { name: 'count' })` read at module top-level (no owner, no subscriber): `console.warn` called with `'[Streem] Signal "count" read outside reactive context'`
    - Anonymous `signal(0)` read at top-level: `console.warn` called with `'[Streem] Signal read outside reactive context'`
    - Signal read inside `createRoot` with active effect: NO warning
    - Signal read inside `computed`: NO warning

    **DX-03 (dev-warnings.test.ts):**
    - `effect(() => {})` called at module top-level (no owner): `console.warn` called with `'[Streem] effect() created without an active owner scope'`
    - `computed(() => 1)` called at module top-level: `console.warn` called with `'[Streem] computed() created without an active owner scope'`
    - `effect()` called inside `createRoot`: NO warning
    - `computed()` called inside `createRoot`: NO warning

    **Cases: input → expected output:**

    signal(0)()                        → 0
    signal(0); count.set(5); count()   → 5
    computed(() => count() * 2); count.set(3); doubled() → 6
    effect runs count() → log.push; count.set(1) → log = [0, 1]
    createRoot(d => { effect(...) }); dispose(); count.set(99) → no re-run
    onCleanup(cb) in effect; count.set(1) → cb fires THEN new run
    signal read outside root → console.warn called once
    effect() outside root → console.warn called once
