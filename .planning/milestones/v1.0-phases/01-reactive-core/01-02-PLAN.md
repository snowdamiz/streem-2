    Create packages/core/src/signal.ts implementing the full public API per CONTEXT.md locked decisions.

    **Signal type (LOCKED by user decision):**
    ```typescript
    export interface Signal<T> {
      (): T
      set(value: T): void
    }
    ```

    `signal<T>(initialValue: T, options?: { name?: string }): Signal<T>`:
    - Creates an internal SignalNode via `createSignalNode(initialValue)`
    - Returns a getter function with `.set()` attached
    - Getter: calls `trackRead(node)` then returns `node.value`
    - Dev warning check in getter (DX-02): if `import.meta.env.DEV` AND `getCurrentSubscriber() === null` AND `getOwner() === null` → `console.warn('[Streem] Signal${label} read outside reactive context. This is likely a snapshot.')` where label is ` "${options.name}"` if name provided, empty string otherwise
    - `.set(value)`: uses `Object.is()` equality check — only notify if value actually changed (avoids spurious re-runs)
    - CRITICAL: Use `if (import.meta.env.DEV)` directly — never assign to intermediate variable, or tree-shaking fails in prod (see research Pitfall 3)

    `computed<T>(fn: () => T): () => T`:
    - Dev warning check (DX-03): if `import.meta.env.DEV` AND `getOwner() === null` → `console.warn('[Streem] computed() created without an active owner scope. This computation will never be automatically disposed (disposal leak).')`
    - Creates internal ComputedNode via `createComputedNode(fn)`
    - Registers the computed node as a child of currentOwner (so it disposes with the owner scope)
    - Returns a getter function that calls `trackRead(node)` then evaluates lazily (checks dirty/pending flag, re-runs fn if needed, returns cached value otherwise)

    `effect(fn: () => void): () => void`:
    - Dev warning check (DX-03): if `import.meta.env.DEV` AND `getOwner() === null` → `console.warn('[Streem] effect() created without an active owner scope. This effect will never be automatically disposed (disposal leak).')`
    - Creates internal EffectNode via `createEffectNode(fn)`
    - Registers the effect node as a child of currentOwner
    - Runs fn() immediately on creation (synchronous first run — see research Open Question 3)
    - Returns a manual dispose function (separate from owner-scope disposal — per CONTEXT.md locked decision)
    - The dispose function: clears the effect's deps, removes it from its owner's children, fires its registered cleanups

    Note on `onCleanup` inside effects: When effect fn() calls `onCleanup(cb)`, that `cb` must fire BOTH before each re-execution AND on owner disposal. This is achieved by registering cleanup callbacks on the EffectNode itself (not just the owner). Before re-running, the effect's node cleanups are fired and cleared; when the owner disposes, the EffectNode disposal fires remaining cleanups. Coordinate with how reactive.ts implements EffectNode.run() to ensure both paths work.
