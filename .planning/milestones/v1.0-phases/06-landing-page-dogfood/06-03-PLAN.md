From packages/dom/src/h.ts — applyProps() namespace routing:
```typescript
// prop:    → routes to el[propName] = value (JS property, not attribute)
// attr:    → routes to el.setAttribute(attrName, value)
// on:      → routes to el.addEventListener(eventName, handler) with onCleanup
// (bare)   → static attribute via setAttribute

// Usage in TSX:
<sl-button prop:variant="primary" prop:size="large" on:click={handler}>
  Label
</sl-button>
// Equivalent to:
// el.variant = "primary"      (property binding)
// el.size = "large"           (property binding)
// el.addEventListener('click', handler)  (direct listener, no Shadow DOM retargeting)
```

From packages/lit/scripts/gen-lit-types.ts — CEM type generation:
```typescript
// The --pkg flag reads: ./node_modules/<pkgName>/custom-elements.json
// Writes to: src/lit-types/lit-elements.d.ts (relative to packages/lit/)
// Run from monorepo root to ensure Shoelace is in root node_modules/:
// pnpm --filter /lit tsx packages/lit/scripts/gen-lit-types.ts --pkg @shoelace-style/shoelace
//
// NOTE: If custom-elements.json is not at root of Shoelace package, check dist/custom-elements.json
// Verify with: ls node_modules/@shoelace-style/shoelace/custom-elements.json

// Generated output format (augments /dom/jsx-runtime):
// declare module '/dom/jsx-runtime' {
//   namespace JSX {
//     interface IntrinsicElements {
//       'sl-button': { 'prop:variant'?: string; 'prop:size'?: string; ... }
//       'sl-badge': { ... }
//     }
//   }
// }
```

Shoelace v2 integration rules:
1. `setBasePath` MUST be called before any Shoelace component JS is imported (done in main.tsx from 06-01)
2. Cherry-pick components: `import '@shoelace-style/shoelace/dist/components/button/button.js'`
3. Do NOT import from main Shoelace entry (bundles entire library)
4. Dark theme is activated via `class="sl-theme-dark"` on `<html>` (done in index.html from 06-01)
5. For sl-button click: use `on:click` (standard DOM click event bubbles correctly from sl-button)
6. For Shoelace-specific custom events (sl-change, sl-input): use `on:sl-change`, `on:sl-input`

Shoelace FOUCE prevention already in global.css (from 06-01):

Create `.github/workflows/deploy-landing.yml`:

```yaml
# Deploy Streem landing page to GitHub Pages
# Triggers on push to main when landing app or packages change.
# Uses OIDC-authenticated GitHub Pages Actions (not the deprecated gh-pages npm approach).
# Source: https://vite.dev/guide/static-deploy#github-pages

name: Deploy Landing to GitHub Pages

on:
  push:
    branches: [main]
    paths:
      - 'apps/landing/**'
      - 'packages/**'
      - '.github/workflows/deploy-landing.yml'
  workflow_dispatch:

# Required permissions for OIDC-based Pages deployment
permissions:
  contents: read
  pages: write
  id-token: write

# Only allow one deployment at a time; cancel in-progress if new push arrives
concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v3
        with:
          version: latest

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: lts/*
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build landing page
        run: pnpm --filter /landing build

      - name: Configure GitHub Pages
        uses: actions/configure-pages@v5

      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v4
        with:
          # Must match Vite build.outDir (default: dist) relative to apps/landing/
          path: apps/landing/dist

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```
