  <action>
    For each landing component, perform the migration:

    **Pattern:**
    1. Extract the CSS from the `<style>{`...`}</style>` block into a `.module.css` file
    2. Convert class names from kebab-case selectors to camelCase (so TS property access works): `.hero-badge` ã `.heroBadge`, `.section-label` ã `.sectionLabel`, etc.
    3. Update the component to: `import styles from './ComponentName.module.css'`
    4. Replace `class="hero-badge"` ã `class={styles.heroBadge}` in JSX
    5. Remove the `<style>{...}</style>` block from the component

    **IMPORTANT rules for CSS extraction:**
    - Keep CSS variable references (`var(--color-accent)` etc.) as-is ã they resolve from global.css
    - Pseudo-selectors (`:hover`, `:not(:defined)`) are handled by referencing the class: `.featureCard:hover` stays `.featureCard:hover` in the module
    - `:global()` wrapping is NOT needed ã these are all class-based selectors
    - For combinator selectors like `.ticker-table th` ã in a CSS module, write as `.tickerTable th` (descendant still works)
    - Exception: `.ticker-skeleton .ticker-row td` ã two local classes together: `.tickerSkeleton .tickerRow td` ã works fine

    **Hero.tsx migration:**
    - Extract all styles from the `<style>` block
    - Class name mapping: `.hero` ã `.hero`, `.container` ã `.container` (but `.container` is a global utility! Keep it as `class="container"` without module ã ONLY move component-specific classes to the module. Global layout classes used everywhere like `.container`, `.section-label`, `.section-title`, `.section-sub` that appear in multiple components should remain as global strings OR extract them to global.css.)

    Actually, looking at the codebase, `.container` is already in global.css. Other "shared" classes like `.section-label`, `.section-title`, `.section-sub` appear in multiple components ã these are candidates for global.css addition OR can be duplicated in each module (CSS Modules are designed for isolation, so duplication is fine for small shared patterns, or they can be added to global.css).

    **Decision (Claude's Discretion):** Move shared presentational classes (.section-label, .section-title, .section-sub) to the global CSS file since they are structural layout patterns shared across sections. Move component-specific classes to each component's module. This avoids module duplication.
