  <name>Task 1: Add triple-slash references to index.ts and configure dts to include ambient declarations</name>

  <action>
The problem: vite-plugin-dts only processes TypeScript source files that are imported via the module graph. Ambient .d.ts files (base-custom-element-types.d.ts, lit-types/lit-elements.d.ts) are never imported — they are pure type augmentations — so they are silently excluded from the rollup output.

Fix strategy: Add `/// <reference path="..." />` triple-slash directives to `src/index.ts`. These directives are TypeScript's mechanism for including type-only files that have no JS counterpart. When vite-plugin-dts sees triple-slash references in the entry point, it includes those referenced files' declarations in the rolled-up output.

Step 1 — Update `packages/lit/src/index.ts`:
Add two triple-slash reference directives at the top of the file (before the export statements):
```typescript
/// <reference path="./base-custom-element-types.d.ts" />
/// <reference path="./lit-types/lit-elements.d.ts" />
```

These must be at the very top of the file (before any imports/exports) to be recognized as triple-slash directives — if placed after other statements they become comments.

Step 2 — Update `packages/lit/vite.config.ts`:
Change `dts({ rollupTypes: true })` to include the src directory so ambient files are discoverable:
```typescript
plugins: [dts({ rollupTypes: true, include: ['src'] })]
```

The `include` option tells vite-plugin-dts to scan the src directory for all .ts and .d.ts files, not just the module graph starting from the entry point.

Step 3 — Verify the changes look correct before building:
Run `npx tsc --noEmit --project packages/lit/tsconfig.json` to confirm no TypeScript errors were introduced by the changes.
