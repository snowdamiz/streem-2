**Change 1 — Remove array spread in propagateDirty:**

Current (line 173):
```typescript
function propagateDirty(source: SourceNode): void {
```

Replace with direct Set iteration. The spread "snapshot to avoid mutation during iteration" comment is no longer needed because:
- The outer `notifying = true` guard in `notifySubscribers()` prevents re-entrant calls from mutating `source.subs` during the loop
- Effects never add themselves to `source.subs` during `runEffect()` — they only clear+rebuild their `deps`, which removes them from old sources and re-adds via `trackRead()`
- Mutation of `source.subs` only happens via `trackRead()` (add) and `source.subs.delete()` (remove), both of which occur while `currentSubscriber` is set — not during outer `propagateDirty` traversal

New code:
```typescript
function propagateDirty(source: SourceNode): void {
  for (const sub of source.subs) {
```

Remove the old `const subs = [...source.subs]` line entirely.

Make the `children` and `cleanups` arrays on Owner lazy (null until first use). Most createRoot() calls in benchmarks never acquire children or register cleanups — they create the owner, run one signal operation, then dispose. Allocating empty arrays eagerly wastes memory for this common pattern.

**Step 1: Update the Owner interface:**

```typescript
export interface Owner {
  parent: Owner | null
  children: Owner[] | null   // null until first child is added
  cleanups: (() => void)[] | null  // null until first cleanup is registered
  disposed: boolean
}
```

**Step 2: Update createRoot() to initialize with nulls:**

```typescript
export function createRoot<T>(fn: (dispose: () => void) => T): T {
  const owner: Owner = {
    parent: currentOwner,
    children: null,   // lazy
    cleanups: null,   // lazy
    disposed: false,
  }

  // Register as child of the current owner (if any)
  if (currentOwner !== null) {
    if (currentOwner.children === null) {
      currentOwner.children = []
    }
    currentOwner.children.push(owner)
  }
  // ... rest unchanged
```

**Step 3: Update disposeOwner() to guard null arrays:**

```typescript
function disposeOwner(owner: Owner): void {
  if (owner.disposed) return
  owner.disposed = true

  // Bottom-up: dispose children first
  if (owner.children !== null) {
    for (const child of owner.children) {
      disposeOwner(child)
    }
  }

  // Fire own cleanup callbacks synchronously
  if (owner.cleanups !== null) {
    for (const cleanup of owner.cleanups) {
      cleanup()
    }
  }

  // Release references
  owner.children = null
  owner.cleanups = null
}
```

**Step 4: Update onCleanup() to lazy-init the cleanups array:**

```typescript
export function onCleanup(fn: () => void): void {
  if (currentEffectCleanupTarget !== null) {
    currentEffectCleanupTarget.cleanupFns.push(fn)
  } else if (currentOwner !== null) {
    if (currentOwner.cleanups === null) {
      currentOwner.cleanups = []
    }
    currentOwner.cleanups.push(fn)
  }
}
```

NOTE: The `OwnerRef` interface in reactive.ts references `cleanups: (() => void)[]` — this is a structural interface used for `ComputedNode.owner` and `EffectNode.owner`. Since Owner now has `cleanups: (() => void)[] | null`, check whether the OwnerRef structural match still holds. In reactive.ts's `createComputedNode` and `createEffectNode`, the owner is only used for type-passing; actual cleanup access is done in owner.ts via disposeOwner(). The OwnerRef interface only requires `cleanups: (() => void)[]` (non-nullable) — but these nodes never call owner.cleanups directly.

To avoid changing OwnerRef and breaking the structural match: update `OwnerRef` in reactive.ts to `cleanups: (() => void)[] | null` as well, keeping the structural subtype valid. The owner field on computed/effect nodes stores the Owner reference for identification, not for direct cleanup access.
