## Decisions Made
- Used public API (`signal()`, `computed()`, `effect()`) outside createRoot rather than internal node constructors — prod build strips DEV warnings so there is no warning overhead, and the public API is what the benchmark should measure
- effect() primitive-only uses `stop()` (the return value of `effect()`) for per-iteration disposal, exactly matching Preact's `cleanup()` return pattern
- Relabeled existing variant to `(with createRoot)` to make the scope explicit and contrast clear
