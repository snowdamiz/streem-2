Add a section documenting what changed from the Phase 9 baseline:

```markdown
## Phase 9.1 Optimizations

These results replace the Phase 9 baseline. The following changes were made to
`/core` between Phase 9 and Phase 9.1:

### Reactive graph improvements (`reactive.ts`)

- **Eliminated array spread in `propagateDirty()`**: The hot path previously
  allocated `[...source.subs]` on every signal write to snapshot subscribers.
  Now iterates the Set directly. The re-entrant guard (`notifying = true`)
  prevents concurrent mutation, making the snapshot unnecessary.

- **O(1) effect deduplication in batch mode**: `batchedEffects` changed from
  `EffectNode[]` (with `Array.includes()` O(n) scan) to `Set<EffectNode>`
  (O(1) `.add()` deduplication).

### Owner tree improvements (`owner.ts`)

- **Lazy array initialization**: `Owner.children` and `Owner.cleanups` are now
  `null` until first use, instead of eagerly allocating empty arrays. Most
  createRoot scopes in the benchmark never acquire children or cleanups,
  so this reduces allocation per iteration.

### Benchmark fairness

- **Primitive-only variants added**: Each suite now measures `/core`
  both with and without a `createRoot` wrapper. The primitive-only variant is
  directly comparable to `@preact/signals-core` which has no owner concept.
  The with-createRoot variant matches real-world usage (and SolidJS's pattern).

### Baseline comparison

| computed re-evaluation | 3,466,920 | [Task 1 result] | [Task 1 result] |
| effect re-run | 3,267,971 | [Task 1 result] | [Task 1 result] |
```
