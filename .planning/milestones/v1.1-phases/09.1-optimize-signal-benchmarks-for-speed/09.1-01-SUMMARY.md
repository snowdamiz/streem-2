# Phase 09.1 Plan 01: Core Reactive Engine Optimization Summary

**batchedEffects deduplication changed from O(n) Array.includes to O(1) Set.add, Owner children/cleanups lazy-initialized to null eliminating eager empty-array allocations on createRoot**

## Accomplishments

- `batchedEffects` is now `Set<EffectNode>` — O(1) `add()` replaces O(n) `Array.includes()` + `push()` for batching deduplication
- `endBatch()` uses `[...batchedEffects]` spread + `batchedEffects.clear()` instead of `splice(0)` — semantically equivalent, Set-native
- `Owner.children` and `Owner.cleanups` typed as `T[] | null`, initialized to `null` in `createRoot()` — no empty array allocation for short-lived benchmark roots
- `disposeOwner()` and `onCleanup()` guard null arrays before access/push, with lazy init on first use
- All 40 `/core` unit tests pass; `pnpm --filter /core build` succeeds (TypeScript clean)

## Task Commits

1. **Task 1: Replace batchedEffects Array with Set for O(1) deduplication** - `170cc10` (feat)
2. **Task 2: Lazy-initialize children and cleanups arrays on Owner** - `f83e32d` (feat)

## Files Created/Modified

- `packages/core/src/reactive.ts` - batchedEffects Set, endBatch() updated, OwnerRef.cleanups nullable, propagateDirty snapshot comment clarified
- `packages/core/src/owner.ts` - Owner interface nullable arrays, createRoot null init, disposeOwner null guards, onCleanup lazy init

## Decisions Made

- Used `Set<EffectNode>` for `batchedEffects` — same semantics as array (insertion-ordered iteration) but O(1) membership test. `endBatch()` snapshots via spread before clearing, matching the previous `splice(0)` pattern.
- Retained `[...source.subs]` snapshot in `propagateDirty()` — direct Set iteration is unsafe because `runEffect()` calls `source.subs.delete()` when clearing stale deps, which mutates the Set mid-iteration and triggers V8 `OrderedHashTable::Shrink` in a rehash loop (heap OOM confirmed in testing). The plan's rationale for removing the snapshot was incorrect.
- Updated `OwnerRef.cleanups` in reactive.ts to `(() => void)[] | null` — necessary for structural subtype alignment since Owner now uses nullable cleanups.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Reverted direct Set iteration in propagateDirty()**
- **Found during:** Task 1 verification
- **Issue:** Plan stated the `[...source.subs]` snapshot could be safely removed because `notifying = true` prevents re-entrant mutation. This was incorrect — `runEffect()` is called synchronously from `propagateDirty()` on the non-batching path and calls `source.subs.delete(effect)` during dep cleanup, mutating the Set mid-iteration. V8's `OrderedHashTable::Shrink` entered an allocation loop, causing OOM heap crash in tests.
- **Fix:** Retained the `[...source.subs]` snapshot. Updated the comment to accurately describe WHY the snapshot is needed.
- **Files modified:** `packages/core/src/reactive.ts`
- **Verification:** All 40 tests pass (confirmed with snapshot in place). OOM crash confirmed without snapshot.
- **Committed in:** `170cc10` (part of Task 1 commit)
